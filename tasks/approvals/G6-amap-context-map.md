# G6 审批：高德上下文地图与 MapLibre 成果地图双模式切换

## 审批状态

- 门禁：G6（高德浏览器地图范围变更）
- 状态：待审批
- 提出日期：2026-07-22
- 用户已确认的体验选择：任务执行期间继续显示高德地图；使用普通道路地图；高德不可用时允许
  回退到现有离线占位图
- 前置决策：[G5 高德 Web 服务研究区校验](G5-amap-web-service-integration.md)
- 批准前限制：只允许评审本文件，不得接入高德 JS API、申请的真实 Key 或安全密钥

## 变更原因

当前 Web 在没有发布成果时不会初始化地图引擎，而是显示一个由 CSS 绘制的流域形状和等待文案。
只有 Publisher 返回合法成果后，Web 才创建 MapLibre 地图，加载完整流域边界和三类 NDVI 瓦片。
因此空闲和任务执行期间的地图区域缺少道路、地名和真实地理语境，视觉上较为单调。

G5 明确限定高德为 Master 后端专用、可降级的地名校验源，并明确不接入高德 JS API/底图。
本变更会增加浏览器到高德 JS API 的新信任边界和一个同源安全代理路径，不能视为 G5 的自然延伸，
必须由独立 G6 门禁批准。

## 拟批准的用户体验

采用“高德上下文地图 + MapLibre 成果地图”的双模式，而不是用高德替换整个 GIS 成果地图。

| 页面状态 | 地图表现 | 失败/降级表现 |
| --- | --- | --- |
| 没有任务 | 显示高德普通道路地图，定位神农溪区域，并显示“等待创建任务” | 高德未配置、加载超时或失败时显示现有离线占位图 |
| 任务执行中且没有发布成果 | 继续显示同一高德道路地图，叠加当前任务短 ID 和“正在生成分析成果” | 高德故障只改变背景地图，不改变任务、Agent 时间线或重试 |
| 任务失败且没有发布成果 | 保留高德道路地图，右侧如实显示失败和安全重试 | 高德故障时保留现有离线占位图和失败证据 |
| Publisher 返回合法完整成果 | 销毁高德地图，创建现有 MapLibre 成果地图 | 按现有瓦片失败/重新加载逻辑处理 |
| Publisher 成果契约非法或不可用 | 不显示高德来掩盖成果错误，继续显示现有“地图图层暂不可用”状态 | 任务和时间线仍可使用 |

高德地图只提供位置上下文，不代表分析结果。地图状态文案必须明确区分“高德位置参考”和
“已发布遥感成果”。

## 拟批准的地图范围

1. 使用高德地图 JS API 2.0 的默认普通道路图层；不加载卫星、路网叠加、实时交通、3D、室内图、
   自定义地图、POI 搜索、路线规划、定位或天气插件。
2. 只把随项目批准流域计算得到的一个固定 WGS84 中心点交给官方 `AMap.convertFrom`，转换类型为
   `gps`，仅用于设置高德地图视野；不发送完整流域、多边形顶点、栅格范围或用户输入。
3. 转换后的 GCJ-02 中心点只存在于当前浏览器地图实例内，不写入源码、Local Storage、
   IndexedDB、PostGIS、Redis、日志、报告、成果或测试夹具。
4. 不在高德地图上叠加 G2 流域边界、Publisher XYZ 瓦片、NDVI、差值、变化分级、面积统计或
   Quality 结论。
5. Publisher 成果仍以批准的 WGS84/Web Mercator 元数据和瓦片进入 MapLibre；G2 数据、坐标、
   日期、反射率修正、算法、质量阈值和校验和边界全部不变。
6. 页面从高德切换到 MapLibre 时必须先销毁高德实例、事件和容器引用，避免两个 WebGL 地图实例
   同时运行；切回空闲态时同样销毁 MapLibre 实例。

## Key、配置与安全代理边界

新增配置与现有后端高德配置完全分离：

| 配置 | 可见范围 | 用途 | 是否允许提交 Git |
| --- | --- | --- | --- |
| `AMAP_WEB_SERVICE_KEY` | 仅 Master | 保持 G5 后端研究区地名校验 | 否 |
| `VITE_AMAP_JS_API_KEY` | 浏览器可见 | 高德 Web端（JS API）应用 Key | 否，只提交空占位 |
| `AMAP_JS_API_SECURITY_CODE` | 仅 Web 服务端 | JS API `securityJsCode` | 否 |
| 高德加载超时 | 可公开 | 默认 5 秒，超时后切换离线占位图 | 是，可提交默认值 |

批准以下控制：

1. 必须在高德控制台单独创建服务平台为“Web端（JS API）”的 Key 和安全密钥；不得把现有
   “Web 服务”Key 当成 JS API Key，也不得让两套 Key 共用同一个环境变量。
2. Web JS API Key 按官方机制由浏览器使用，但真实值只进入 Git 忽略的本地 `.env`；源码、
   `.env.example`、测试、截图和文档中只能出现空值或明显占位符。
3. `securityJsCode` 不得使用 `VITE_` 前缀，不得进入浏览器 bundle、HTML、公共配置响应、日志或
   Git。Web 服务端通过固定同源路径 `/_AMapService/*` 保存并附加安全密钥。
4. 安全代理只允许高德官方文档规定的固定路径、固定 HTTPS 上游和所需的只读方法；禁止接受
   用户提供的 URL、Host、协议、端口、重定向目标或任意请求头，不能演变成开放代理或 SSRF。
5. 代理错误使用固定中文/结构化结果，日志不得包含安全密钥、完整查询串或上游响应正文；需要
   配置连接/读取超时、响应上限和并发上限。
6. `/_AMapService/*` 是浏览器地图鉴权所需的同源基础设施路径，不纳入业务 OpenAPI，不新增任务、
   Agent、数据或成果能力；宿主仍只绑定回环地址。
7. 高德 JS API 加载、中心点转换或代理故障不得成为 Master `/api/v1/config/readiness` blocker，
   不得禁用“创建监测任务”按钮，也不得改变任务终态。
8. Web JS Key 应在高德控制台采用可用的来源/域名限制和额度告警；交付或曝光后单独轮换，不影响
   `AMAP_WEB_SERVICE_KEY`。

高德官方说明，2021-12-02 后创建的 Web端（JS API）Key 必须配合安全密钥使用，并强烈建议把
安全密钥保存在服务器端、通过代理转发。G6 不批准把 `securityJsCode` 明文放入前端的便捷模式。

## 外部数据、隐私和许可边界

- 浏览器使用高德地图时会直接加载高德在线脚本和地图资源，最终用户的网络地址、User-Agent 和
  高德为提供服务所需的信息进入新的第三方边界；页面不得宣称高德地图是离线或本地数据。
- 除固定 WGS84 中心点外，不向高德发送用户提示、任务 ID、attempt、correlation ID、Agent 状态、
  流域几何、栅格、统计、质量结果、报告或其他项目数据。
- 不抓取、预取、下载、缓存、镜像、截图保存、转存或再分发高德脚本、瓦片、道路、地名、坐标
  转换结果或其他服务内容；不把高德地图内容写入 PDF 报告或离线回退资源。
- 必须保留高德官方 Logo、审图号、版权和 attribution，不用项目面板、遮罩或控件覆盖。
- 答辩现场可以在应用中实时展示高德地图；如果要把包含高德地图的截图或录屏放入答辩材料、
  仓库或其他长期证据，应先由项目所有者依据当时的服务协议、账号用途和授权条件确认，不在 G6
  中预先声明拥有复制或再分发权利。
- 高德协议、账户状态、额度或授权条件变化时，应关闭 JS API 配置并使用离线占位图，不得绕过
  鉴权、伪造来源或抓取瓦片维持展示。

## 加载、生命周期与降级规则

1. 按官方 React 指引把高德 Loader、地图创建和 `destroy()` 清理封装在独立组件内；不在全局
   反复插入脚本，不让卸载后的 Promise 回调重新创建地图。
2. JS API 只允许按官方方式在线加载，不把脚本下载到仓库、镜像、Service Worker 或浏览器离线
   缓存中，也不把高德代码与项目 bundle 混合转存。
3. 没有 JS API Key、安全代理未配置、Loader 拒绝、转换失败、鉴权失败或 5 秒内未达到可用状态
   时，组件进入稳定的“离线地图”状态并渲染现有 `MapPlaceholder`。
4. 降级只在当前页面内记录提供商无关的状态码，例如 `NOT_CONFIGURED`、`LOAD_TIMEOUT`、
   `LOAD_FAILED`；不保存高德原始错误对象、URL、响应或坐标。
5. 页面重试高德加载必须由明确的用户操作或下一次正常组件挂载触发，不做无限重试、后台轮询或
   指数级脚本注入，避免额度和资源失控。
6. 高德地图加载成功后只创建一个地图实例；任务状态更新不得重复创建实例。收到合法 publication
   后销毁高德并创建一个 MapLibre 实例。
7. 无论高德状态如何，现有任务创建、SSE/轮询、刷新恢复、失败、重试、Publisher 成果和 PDF
   下载行为保持不变。

## 明确不批准的方案

- **不全面替换 MapLibre：**现有成果和流域采用 WGS84/Web Mercator，高德采用 GCJ-02；直接迁移
  会引入栅格偏移、重投影和既有验收失效风险。
- **不把高德瓦片 URL 填入 MapLibre：**不依赖非官方瓦片地址，不绕过官方 JS API、Logo、鉴权
  或服务条款。
- **不把 Publisher XYZ 瓦片填入高德 TileLayer：**即使接口形式支持第三方瓦片，现有瓦片没有
  为 GCJ-02 底图做显示变换，不能用“看起来接近”替代空间精度。
- **不转换完整流域或成果：**只转换一个固定中心点，不分批上传完整边界，不新增 GCJ-02 成果副本。
- **不增加高德搜索和位置业务：**不加入 POI、路线、定位、天气、交通、卫星图或第二 GIS 场景。
- **不缓存高德供离线答辩：**无网时显示项目自有占位图，真实分析继续使用已批准离线栅格。
- **不降低现有成果门禁：**高德背景不能让未完成、失败、跨任务或 Quality 未通过的成果提前出图。

## 考虑过但未采用的替代方案

### 仅美化现有占位图

优点是无新 Key、第三方脚本、坐标和许可风险；缺点是无法提供用户要求的真实道路和地名语境。
作为高德降级态保留，不作为主要空闲态。

### 高德承担所有状态和成果图层

界面看起来统一，但需要重新处理 WGS84/GCJ-02 显示关系，改变 Publisher 瓦片、边界和验收证据，
收益与“改善空闲地图”目标不成比例，因此拒绝。

### 在 MapLibre 中使用非官方高德瓦片地址

改动表面较小，但缺少受支持的官方接入、鉴权和生命周期边界，并增加协议与稳定性风险，因此拒绝。

## 批准后的验收条件

### 功能与界面

- 没有任务和任务执行期间都显示高德普通道路地图，状态文案和任务短 ID 正确；任务状态更新不会
  重建地图实例。
- 任务完成并获得合法 publication 后，高德实例已销毁，现有 MapLibre 三类 NDVI 图层、流域边界、
  日期、图例、attribution、瓦片错误和重载行为全部保持通过。
- 任务失败且没有成果时仍诚实显示失败/重试，高德背景不产生“成果已发布”的误导。
- 宽屏与窄视口均保留高德 Logo/版权、任务文案、地图操作和控制栏可读性；键盘焦点和无障碍名称
  不退化。

### 降级与稳定性

- JS API Key 或安全代理未配置时不发出高德请求，直接显示离线占位图，任务创建按钮仍可用。
- Loader 超时、网络断开、鉴权失败、坐标转换失败和组件卸载竞态均在 5 秒内进入/保持可恢复的
  离线占位状态；没有白屏、无限重试、未处理 Promise 或浏览器控制台错误。
- 高德从可用变为不可用不改变已有任务、Agent 时间线、PostGIS、分析成果或报告；刷新后按当前
  配置重新判定上下文地图状态。
- 页面切换和多次任务运行中，高德与 MapLibre 实例、监听器、Canvas/WebGL 上下文和 Loader 调用
  数量保持有界。

### 安全、网络和数据

- `AMAP_WEB_SERVICE_KEY`、Web JS Key 和 `securityJsCode` 三者不混用；真实值不进入 Git、日志、
  数据库、Redis、报告、测试产物或验收截图。
- 生产式浏览器检查确认 `securityJsCode` 不在 HTML、JavaScript bundle、source map、公共配置响应
  或浏览器网络请求中；代理无法访问任意上游、私网或非批准路径。
- 高德请求中不存在用户提示、任务/关联标识、完整流域、成果或报告；只允许官方加载/地图资源和
  一个固定中心点转换。
- 不持久化高德脚本、瓦片、道路、地名、坐标转换结果或原始错误；离线回退不读取缓存高德内容。
- 浏览器网络验证只访问批准的高德官方域名、同源安全代理、Master 和 Publisher；无未知第三方。

### 自动化与真实浏览器

- Web 组件测试覆盖未配置、成功加载、任务期间复用、5 秒超时、失败回退、卸载竞态、成果切换和
  MapLibre 既有地图行为。
- TypeScript、ESLint、Vitest、Vite build、Compose 配置、依赖审计和凭据扫描通过。
- 默认 Playwright E2E 使用假 Loader 或路由拦截，不访问真实高德、不消耗额度，冷/温缓存旅程仍
  通过。
- 使用专用可轮换 Web端（JS API）Key 完成一次显式真实浏览器冒烟：普通道路地图可见、中心区域
  正确、官方标识未被覆盖、控制台干净；随后断开高德网络，验证 5 秒内回退且任务完整完成。
- 真实冒烟证据只记录时间、状态、耗时、脱敏错误码和网络域名集合，不保存高德地图截图、Key、
  安全密钥、原始响应或坐标结果。

## 与 G5 的关系

G6 若获批准，只对 G5 的以下限制做最小补充：

- 允许 Web 在没有发布成果时使用高德 JS API 普通道路地图；
- 允许一个独立、浏览器可见但不提交的 Web JS Key；
- 允许 Web 同源固定安全代理和一个固定中心点的临时坐标转换。

G5 的其余边界继续有效：`AMAP_WEB_SERVICE_KEY` 仍只注入 Master；高德地名校验仍可降级；不缓存
高德原始响应；不替代 G2 数据；不扩大分析地区；不改变 Agent 链、质量门禁或 Publisher 成果。
