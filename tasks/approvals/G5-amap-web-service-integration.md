## 需要批准的决策

批准将高德 Web 服务 API 作为**后端专用、可降级的研究区校验源**接入 Master Agent：

1. Master 先用本地允许别名确认请求属于已批准的“神农溪生态变化监测”场景，再通过高德
   地理编码或限定巴东县 `adcode` 的 POI 搜索做在线交叉校验；只发送固定规范名称和
   `adcode`，不发送用户完整提示词、任务/关联标识或影像/成果信息。
2. 高德校验成功只增加“在线位置校验通过”证据；Data、Analysis、Quality、Publisher 仍以
   G2 本地流域、影像、日期、校验和和计算成果为唯一 GIS 依据。
3. 明确指向其他地区的请求返回中文范围错误，不再被静默绑定到神农溪；多结果或歧义结果
   不得标记为验证通过。
4. 高德未配置、超时、限流或不可用时，规范内神农溪任务显示“在线位置校验已降级”并继续
   本地离线主链；高德不是核心就绪阻塞项。
5. `AMAP_WEB_SERVICE_KEY` 只注入 Master，永不进入浏览器、公共 API、日志、PostGIS、Redis、
   报告或 Git；交付前轮换当前已暴露的测试 Key。
6. 固定访问 `https://restapi.amap.com`，禁止生产环境自定义目标域名；限制连接/读取超时、
   响应大小和重定向，严格验证 JSON 与高德状态码。
7. 不抓取、缓存、持久化、下载、镜像或再分发高德 POI、行政区边界、坐标、底图及原始响应；
   只保存本系统生成的提供商无关结论、检查时间、耗时和脱敏原因码。
8. 不接入高德 JS API/底图、路线规划、天气、生活 POI 或第二个 GIS 场景；现有 MapLibre
   WGS84/Web Mercator 成果地图保持不变，避免与高德 GCJ-02 坐标直接混用。
9. 不新增公开路由、数据库迁移或基础设施服务；沿用既有任务事件、健康/就绪语义和重试机制。

## 为什么采用这个方案

当前 Master 对任意合法文本都会创建任务并绑定固定神农溪流域。高德地理编码/POI 搜索可为
中文地名提供独立行政区交叉校验，从而减少“用户意图是其他地区，但系统仍输出神农溪成果”的
误导风险。把它设计成可选增强，可以保留答辩所需的离线稳定性，也不会削弱已批准遥感数据和
质量证据的权威性。

## 明确不采用的方案

- **不以高德行政区边界替代流域边界：**行政区与水文流域不是同一空间对象。
- **不以高德底图替代 MapLibre：**现有成果使用 WGS84/Web Mercator，高德使用 GCJ-02；
  直接叠加可能偏移，且会增加在线依赖与授权风险。
- **不缓存高德结果供答辩离线使用：**官方协议限制抓取、缓存和脱离服务使用相关内容。
- **不加入路径规划或附近生活 POI：**与生态变化监测核心旅程无关，会扩大 MVP 范围。

## 验收证据

- 假高德服务覆盖成功、零/多结果、认证、限流、超时、重定向、超大响应、畸形 JSON 和脱敏。
- 显式真实冒烟返回 `infocode=10000`，输出不包含 Key 或原始响应。
- 同一 G2 任务在高德在线与高德不可用两种条件下均完成，分析/质量成果一致，只是校验证据不同。
- 明确非神农溪请求被诚实拒绝，模糊请求不显示“高德已验证”。
- 浏览器、日志、数据库、Redis、报告、Git 和测试产物中均不存在高德 Key 或原始服务数据。
